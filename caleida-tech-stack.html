Technology Stack

Caleida The platform behind Cal — an AI “mate” with a conversational layer, a proactive nudge engine, a household mental-load dashboard, and memory that learns each household over time.

What makes it distinct

Three things shape every decision

Caleida isn’t a to-do app with a chatbot attached. Three properties pull the architecture in a specific direction — and the data is unusually sensitive.

01
A voice that’s protected
Warm roast, never shame. Tone is a product requirement, so it’s enforced inside the pipeline.
02
Proactive intelligence
Cal acts unprompted. A background engine watches household state and decides when to speak.
03
Earned memory
He gets smarter about one household over time — structured state plus semantic recall.

Client — mobile

iOS & Android app

Where Caleida really lives — push-driven, always-on, in the pocket. Two routes, depending on team size and how deep you go into OS features.

Recommended

Cross-platform — React Native + Expo

One TypeScript codebase for both stores.
  • OTA updates — ship copy and tone tweaks without an app-store cycle (key when voice is the product).
  • EAS Build & Submit — managed build and release pipeline.
  • Native modules when needed; one team, fast iteration.
Best fit for a small team shipping to iOS + Android at once.

Fully native

Two codebases, maximum platform fidelity.
  • iOS — Swift + SwiftUI.
  • Android — Kotlin + Jetpack Compose.
  • Richest OS integrations (widgets, Live Activities, Siri / Assistant) — but double the build and skill set.
Flutter (Dart) is a credible single-codebase middle ground if the team prefers it.

The core

Cal’s brain, in detail

This is the heart of the system. Three moving parts, plus the guardrail that protects the voice.

01 The language model

The voice is the moat — warm roast, never cold shame — and that lives or dies on instruction-following and tone control. Claude is strong here, handles long context for carrying household memory into a conversation, and supports native tool use. Keep it behind an interface so routine work (intent, classification) routes to a smaller, cheaper model and the top model is reserved for voice-critical generation — this is also your main cost lever.

02 Orchestration & tools

The Vercel AI SDK (TypeScript) or LangGraph (Python) manages tool-calling and multi-step flows. Cal’s tools are functions that hit your own services — and tool use is what turns talk into action. Principle 03, “noticing, not outsourcing,” is enforced by which tools exist and how they’re framed.

book appointmentset reminderbuild shopping listmark task ownedlog a win

03 Memory — the earned-intelligence loop

  • Structured state in Postgres — the authoritative picture of the household and load distribution; this is also what the dashboard renders.
  • Semantic memory in pgvector — patterns, preferences, and context retrieved per conversation.
  • Rolling summaries — compress history so you’re not paying to resend everything each turn.

04 The proactive engine & tone guardrail

A scheduled background process evaluates household state, detects signals, and decides whether Cal should reach out. It’s a hybrid: deterministic rules decide that a nudge is warranted; the LLM phrases it in voice. That keeps nudges reliable and cheap. Because a message that could make a dad defensive or deflated is wrong (principle 05), a lightweight check rewrites or rejects anything shaming before it ships.

Economics

Keeping cost & ops lean

The dominant variable cost is LLM tokens. Three levers keep it sane.

  • Model routing — a small model for classification and intent; the top model only for voice-critical generation.
  • Memory compression — rolling summaries and retrieval instead of resending full history.
  • The rules / LLM split — deterministic logic decides when to nudge, so tokens are spent only on phrasing.

Start fully managed (Supabase + Render + Upstash) to keep monthly burn low and headcount on the product, not the platform. Re-platform onto AWS or GCP when scale, cost, or compliance makes the operational weight worth it.

In one line

Expo / React Native app → NestJS backend → Postgres + pgvector + Redis → a Claude-powered agent with a rules-driven proactive engine and layered memory → push / email delivery → PostHog, Sentry & Langfuse for observability — managed-first, privacy by design.
